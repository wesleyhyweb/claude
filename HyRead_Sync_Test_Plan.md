# HyRead 電子書閱讀器 — 跨平台同步測試計劃

## 1. 概述

| 項目 | 內容 |
|------|------|
| 產品名稱 | HyRead 電子書閱讀器 |
| 測試範疇 | 跨平台同步功能（閱讀進度、書籤、手寫筆記） |
| 涵蓋平台 | iOS / Android / 桌面程式 / 電子紙閱讀器 |
| 文件版本 | 1.0 |
| 建立日期 | 2026-04-16 |

---

## 2. 測試目標

- 驗證閱讀進度、書籤、手寫筆記可在所有平台間正確同步
- 確認同步資料的完整性，不發生遺失或損毀
- 測試離線操作後恢復網路時的同步行為
- 確認多裝置同時操作時的衝突處理機制

---

## 3. 測試平台與裝置

| 平台 | 測試裝置 | 系統版本需求 |
|------|---------|------------|
| iOS | iPhone / iPad | iOS 16+ |
| Android | Android 手機 / 平板 | Android 10+ |
| 桌面程式 | Windows PC / Mac | Windows 10+ / macOS 12+ |
| 電子紙閱讀器 | HyRead Gaze 系列 | 最新韌體 |

> 所有測試使用**同一帳號**登入四個平台。

---

## 4. 測試環境

| 環境 | 說明 |
|------|------|
| 正常網路 | Wi-Fi 或 4G/5G，作為基準測試環境 |
| 弱網路 | 限速至 1 Mbps，驗證同步穩定性 |
| 離線 | 飛航模式，測試本地儲存與延遲同步 |

## 5. 測試用例

### 5.1 閱讀進度同步

| 編號 | 測試項目 | 前置條件 | 測試步驟 | 預期結果 | 結果 |
|------|---------|---------|---------|---------|------|
| TC-S01 | 單向進度同步 | 四平台同帳號登入，書籍已下載 | 1. 在平台 A 開啟書籍並閱讀至特定頁碼<br>2. 等待同步完成<br>3. 在平台 B 開啟同一本書 | 平台 B 顯示同步提示或自動跳轉至平台 A 的閱讀位置 | |
| TC-S02 | 四平台進度一致性 | 同 TC-S01 | 1. 在 iOS 閱讀至第 30 頁<br>2. 依序於 Android、桌面、電子紙確認進度 | 所有平台顯示相同閱讀位置 | |
| TC-S03 | 同步延遲時間 | 網路正常 | 1. 在平台 A 更新進度<br>2. 立即切換至平台 B 並計時 | 同步完成時間 ≤ 30 秒 | |
| TC-S04 | 多裝置競爭寫入 | 兩裝置同時開啟同一本書 | 1. 在平台 A 閱讀至第 50 頁<br>2. 同時在平台 B 閱讀至第 80 頁<br>3. 確認兩裝置最終同步結果 | 以最後更新的進度為準，資料不損毀，不發生錯誤 | |
| TC-S05 | 弱網路下進度同步 | 網路限速至 1 Mbps | 1. 在平台 A 更新進度<br>2. 切換至平台 B 查看結果 | 同步最終成功完成，資料正確 | |

## 7. 缺陷優先級定義

| 等級 | 說明 | 範例 |
|------|------|------|
| P1 Critical | 同步完全失效，資料遺失 | 進度/書籤/筆記完全未同步 |
| P2 High | 同步不完整或資料損毀 | 書籤部分遺失、筆記內容錯誤 |
| P3 Medium | 同步延遲過長或偶發失敗 | 同步超過 2 分鐘、需手動重試 |
| P4 Low | 同步成功但體驗問題 | 同步提示訊息顯示不正確 |

---

## 8. 測試完成標準

- [ ] 所有 P1 缺陷已修復並通過回歸測試
- [ ] 所有 P2 缺陷已修復或有明確修復計劃
- [ ] 跨平台同步矩陣所有組合均已測試
- [ ] 同步測試用例通過率 ≥ 95%
- [ ] 無已知資料遺失或損毀風險

---

## 9. 風險與假設

| 風險 | 影響 | 緩解措施 |
|------|------|---------|
| 電子紙不支援手寫輸入 | 筆記在電子紙只能顯示，無法編輯 | 確認規格，筆記同步驗證改為唯讀確認 |
| 同步衝突機制未明確定義 | 競爭寫入測試預期結果不明確 | 事前與開發團隊確認衝突解決策略 |
| 電子紙裝置取得困難 | 電子紙相關測試覆蓋不足 | 協調向廠商借測裝置 |
